# Quantile Binning for Microbiome ML

EuroBioC2026 poster project — Rasmus Hindström, Juho Pelto, Tuomas Borman & Leo Lahti  
Department of Computing, University of Turku

## Overview

Standard microbiome count data is zero-inflated and compositional, which makes feature engineering a real choice. This project benchmarks **zero-aware quantile binning** — a novel ordinal encoding — against three established transformations:

| Transformation | Key property |
|---|---|
| Relative abundance (TSS) | Compositional; zeros stay zero |
| rCLR | Log-ratio; zero-robust, no pseudocount |
| Presence/absence (P/A) | Binary; maximum zero-tolerance |
| **Quantile binning** | Ordinal; zeros → bin 0, non-zeros split into `n_bins` equal-count quantile groups per feature |

The binning transformation separates the "absent" state from all "present" states before discretising the abundance gradient. The number of bins (`n_bins`) is treated as a tunable hyperparameter, selected by inner cross-validation for each dataset × classifier combination.

**Research question:** Does zero-aware quantile binning deliver competitive or superior classification performance compared to the standard transformations across a broad collection of curated case/control microbiome datasets?

## Pipeline design

The pipeline is implemented with `{targets}` and uses dynamic branching to parallelise over a grid of **(dataset × transformation × classifier)** combinations. The branching is defined in `_targets.R` via `tidyr::expand_grid()` over the three axes.

### Global parameters

All tuning knobs live at the top of `_targets.R` and propagate downstream automatically:

| Parameter | Default | Description |
|---|---|---|
| `min_class_n` | 50 | Minimum samples per class; datasets below this are dropped |
| `n_bins_grid` | `c(2, 3, 5, 8, 13, 21, 34, 55)` | Fibonacci-ish grid of candidate bin counts |
| `cv_folds` | 5 | Folds for both outer and inner CV |
| `random_seed` | 42 | Controls train/test split; fixed across all models for fair comparison |

### Data

A single `.Rdata` file (`data/tses_85_280126.Rdata`) holds 85 curated `TreeSummarizedExperiment` (TSE) objects from published case/control microbiome studies. Before modeling, each dataset is:

1. **Filtered** — dropped if either class has fewer than `min_class_n` samples (`R/load_data.R`)
2. **Agglomerated** to genus level via `mia::agglomerateByRank()`
3. **Transformed** — TSS, rCLR, and P/A assays added up front; binning is applied inside the CV loop to avoid leakage

### Transformations

Implemented as thin wrappers around `mia::transformAssay()` in `R/transformations.R`:

- `transform_tss()` — relative abundance (TSS)
- `transform_rclr()` — robust centred log-ratio (zero-safe, no pseudocount)
- `transform_pa()` — presence/absence (0/1)
- `transform_binning(n_bins)` — zeros → 0, non-zeros → quantile bins 1…n_bins, applied per feature

TSS, rCLR, and P/A are pre-computed once on the full dataset (no leakage risk). Binning is applied separately inside each CV fold train/test split because `n_bins` is tuned.

### Modeling and binning hyperparameter tuning

`R/modeling.R` wraps `mikropml::run_ml()` for two classifiers:

- **Random Forest** (`ranger`)
- **Regularised regression** (`glmnet` — LASSO/ElasticNet)

For each (dataset, transformation, classifier) triplet, `evaluate_transformation()` dispatches to:

- **Standard transformations**: one `run_ml()` call on the pre-computed assay.
- **Binning**: `evaluate_binning()` iterates over every candidate `n_bins`, runs `run_ml()` with the **same seed** on each (guaranteeing identical train/test splits), picks the `n_bins` with the best inner-CV AUC, and returns that model's held-out test metrics alongside the full tuning trace (`selected_n_bins` column).

Performance metric for selection: inner-CV AUC. Reported metrics: **Accuracy** and **Kappa**.

## Results report

`analysis/report.qmd` renders a self-contained HTML report that reads directly from the targets store.

  report.qmd         # Quarto results report (rendered by tar_quarto())
